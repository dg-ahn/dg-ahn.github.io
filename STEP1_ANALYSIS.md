### 지렁이 게임 규칙

#### 1. 조작

- 방향키 `↑ ↓ ← →` 또는 `W A S D`로 이동한다.
- 지렁이는 자동으로 계속 전진한다.
- 현재 진행 방향의 정반대로는 즉시 이동할 수 없다. 예를 들어 오른쪽 이동 중 왼쪽 입력은 무시한다.
- `Space` 키로 일시정지하거나 다시 시작한다.

#### 2. 점수

- 먹이를 먹으면 10점을 얻는다.
- 먹이를 먹을 때마다 지렁이의 몸길이가 1칸 늘어난다.
- 먹이를 먹은 횟수에 따라 이동 속도가 증가한다.
- 먹이 5개마다 속도가 1단계 증가하며, 최대 속도 제한을 적용한다.
- 최고 점수는 브라우저 또는 기기에 저장한다.

#### 3. 충돌

다음 상황을 충돌로 판정한다.

- 지렁이의 머리가 게임 영역의 벽에 닿은 경우
- 지렁이의 머리가 자신의 몸에 닿은 경우

충돌이 발생한 순간 이동과 점수 계산을 중지한다.

#### 4. 게임 오버

충돌하면 게임 오버 화면을 표시한다. 게임 오버 화면에는 최종 점수, 최고 점수, 먹은 먹이 개수, 다시 시작 안내 또는 버튼을 보여준다.

#### 5. 재시작

- `Enter` 또는 `R` 키를 누르거나 다시 시작 버튼을 클릭한다.
- 지렁이의 위치, 길이, 방향, 이동 속도를 초기화한다.
- 현재 점수는 0점으로 초기화하고 최고 점수는 유지한다.
- 새 먹이는 지렁이의 몸과 겹치지 않는 위치에 생성한다.

`Games` 메뉴와 섹션 이름·순서는 자율 선정한다.

## 가장 안전한 첫 루프

**루프 1: 요구사항·콘텐츠 계약 확정**을 추천한다.

이 루프는 파일을 변경하거나 외부 시스템에 인증하지 않고도 진행할 수 있으며, 현재 확인된 프로필과 미기재 내용을 분리한다. 특히 개인 프로필 사이트는 잘못된 경력·연락처·외부 링크를 공개하는 위험이 있고, 게임은 규칙이 불명확하면 구현·테스트가 반복되기 쉽다. 사람 확인으로 공개 범위와 기본 콘텐츠가 확정되었으므로, 남은 자율 선정 항목만 구현 전에 내부 설계로 고정하면 된다.

1. `Home`, `Profile`, `Work` 또는 `Projects`, `Games`의 최소 정보 구조
2. `Games` 메뉴의 위치와 모바일 메뉴 동작 원칙
3. 프로필에 공개할 문구와 이메일 링크
4. 확정된 지렁이 게임 규칙을 구현 설계와 검증 기준으로 사용
5. 랜덤 적의 수, 이동 속도·범위, 충돌 시 결과는 별도로 자율 설계

이번 요청에서는 위 내용을 분석 문서로만 남겼으며, 코드·테스트·push·배포는 수행하지 않았다.

## 보안 주의사항

- `github_token.txt`는 파일명만 기록했으며 내용은 읽지 않았다.
- 인증이 필요한 작업에서만 토큰 파일을 사용한다.
- 토큰 값을 출력, 로그, 코드, 문서, Git에 남기지 않는다.
- 현재 작업 디렉터리는 Git 저장소로 확인되지 않았으므로, 실제 저장소 구조와 배포 설정은 다음 단계에서 읽기 전용으로 확인해야 한다.
